# RAG Backend — Functional Test Suite

Isolated functional tests for `rag-backend/` (FastAPI + FAISS). They verify every
endpoint and every pipeline stage — ingestion, chunking, embeddings, retrieval,
prompt building, streaming chat, validation & fallback — **on its own**.

- **The remote LLM is mocked** for all core-logic tests (deterministic, offline, no
  GPU cost). Chunking / embeddings / FAISS run the **real** code.
- **One skippable smoke test** (`SM1`) hits the live vLLM to confirm wiring.
- **Fully isolated:** storage is redirected to a throwaway temp dir and the singleton
  store is reset between tests, so the real `data/index/` is never touched. Self-cleaning
  and re-runnable.

See [TEST_PLAN.md](TEST_PLAN.md) for the full matrix and [TEST_RESULTS.md](TEST_RESULTS.md)
for the latest run (actual vs. ideal, PASS/FAIL/PARTIAL). `TEST_RESULTS.md` is
**generated by the run itself**.

## What's covered
| Area | Tests |
|------|-------|
| Health | H1 (ok + reports LLM), H2 (degrades when LLM down) |
| Ingestion | I1 (valid→indexed+persisted), I2 (non-PDF 415), I3 (corrupt), I4a (empty 400), I4b (no-text 422), I5 (list, session-scoped), I6 (delete), I7 (missing session 400) |
| Chunking | CK1 (size+overlap), CK2 (short) |
| Embeddings | E1 (deterministic), E2 (L2-normalized), E3 (dim/dtype), E4 (empty) |
| Retrieval | R1 (relevant), R2 (no-match threshold), R3 (cosine), R4 (session scoping) |
| Prompt | PB1 (context+grounding rule), PB2 (labels/empty) |
| Chat (mocked) | C1 (stream+metadata), C2 (SSE order), C3 (grounded prompt) |
| Validation/fallback | V1 (422), V2 (400), V3 (no_context), V4 (LLM fail), V5 (mid-stream fail), V6 (bad payload) |
| Smoke (real LLM) | SM1 (skippable) |

## Requirements
Python 3.11; the backend's deps (`fastapi`, `faiss-cpu`, `sentence-transformers`,
`pypdf`, `openai`, `pydantic`), plus test-only `pytest` and **`fpdf2`**
(`pip install fpdf2`) — used solely to synthesize fixture PDFs with known content.
The MiniLM embedding model is used for real; it is already cached locally, so tests
run offline.

## Configure
No `.env` is needed for the mocked tests — `conftest.py` sets temp `INDEX_DIR` /
`UPLOADS_DIR` and dummy LLM settings automatically. For the **real-LLM smoke test**,
point at a live server (resume JarvisLabs first — see
`llm-serving/llm-serving/deploy/RESUME_RUNBOOK.md`):

```bash
export LLM_BASE_URL=https://<host>.notebooksn.jarvislabs.net/v1
export LLM_API_KEY=<your vllm key>
export LLM_MODEL=qwen
```

## Run
```bash
# from the repo root — full suite (real-LLM smoke auto-skips)
python -m pytest tests/backend/ -v

# a single area
python -m pytest tests/backend/test_retrieval.py -v
```
First run loads the embedding model, so allow a couple of minutes.

### Run / skip the real-LLM smoke test
- **Skipped by default** (and auto-skips if the endpoint is unreachable).
- **To run it:** set the `LLM_*` vars above, then:
  ```bash
  RUN_REAL_LLM=1 python -m pytest tests/backend/test_smoke_real_llm.py -v
  ```
- **To explicitly exclude the marker:** `python -m pytest tests/backend/ -m "not real_llm"`
