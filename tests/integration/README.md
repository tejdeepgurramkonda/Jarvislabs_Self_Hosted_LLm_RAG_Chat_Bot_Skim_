# RAG System — Integration Tests

Tests the **seams between** the three services with **real requests, real network,
real streaming — no mocks** (mocks were for the isolated backend suite in
`tests/backend/`). See [TEST_PLAN.md](TEST_PLAN.md) for every test and its ideal
result; [TEST_RESULTS.md](TEST_RESULTS.md) is generated by each run.

## What's covered
| Group | Tests | Kind |
|-------|-------|------|
| SEAM 1 — Backend ↔ vLLM | S1.1–S1.5 | pairwise integration |
| SEAM 2 — Frontend ↔ Backend (API level) | S2.1–S2.4 | pairwise integration |
| FULL CHAIN — FE→BE→AI→back | F1–F3 | end-to-end (API level) |
| CROSS-CUTTING — streaming/schema/UX | X1–X3 | end-to-end (API level) |

**Deferred: real-browser layer.** The frontend↔backend seam is tested at the API
level by emulating the browser exactly (Origin header, `X-Session-Id`, SSE reading —
matching `skim-frontend/src/api.js`). A real-browser Playwright test was **not**
included: Playwright isn't installed and needs `@playwright/test` + a browser
download (heavy) plus the frontend dev server running. Can be added on request.

## What must be running
| For | Requirement |
|-----|-------------|
| All live tests | Backend up: from `rag-backend/`, `uvicorn app.main:app --port 8090` |
| LLM tests (S1.1/1.2/1.5, F1/F3, X1–X3) | JarvisLabs vLLM up + `rag-backend/.env` pointing at it (see `llm-serving/.../deploy/RESUME_RUNBOOK.md`) |
| S1.3 (bad key) | vLLM reachable (to get a real 401) |
| S1.4 (unreachable), and pure API tests | backend only |
| Frontend dev server | **not required** (API-level suite) |

Tests that need an unavailable service **skip cleanly** (they don't fail) with a
message telling you what to start.

## Configure
Copy and fill `tests/integration/.env` (secrets are yours to fill):
```
BACKEND_URL=http://localhost:8090
FRONTEND_ORIGIN=http://localhost:5173
# Optional — for the bad-key seam test. If omitted, reused from rag-backend/.env:
# VLLM_BASE_URL=https://<host>.notebooksn.jarvislabs.net/v1
# VLLM_API_KEY=<key>
```
vLLM creds fall back to `rag-backend/.env` (`LLM_BASE_URL`/`LLM_API_KEY`) then
`tests/vllm/.env`, so usually you only need `BACKEND_URL` here.

## Run
```bash
pytest tests/integration/ -v
```
- Errors are recorded per-test; one failure never halts the run.
- `TEST_RESULTS.md` is written at the end (summary + per-test + a perf table).
- Data is throwaway: unique session IDs, documents deleted in teardown; negatives use
  an isolated temp index. Your real `data/index/` is untouched.

## Cost
These hit the real LLM (a handful of calls). **Pause the JarvisLabs instance when
you're done** (`jl pause <id> --yes`).
